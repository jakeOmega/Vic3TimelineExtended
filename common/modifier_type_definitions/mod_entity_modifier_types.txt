# mod_entity_modifier_types.txt
#
# Per-entity registrations of vanilla dynamic-modifier patterns for the mod's
# introduced goods, buildings, ships, units, institutions, laws, and other
# entities. The Clausewitz engine requires each parameterized modifier name
# (e.g. goods_output_<good>_mult, building_<b>_throughput_add,
# ship_battle_against_ship_type_<s>_<axis>_<add|mult>, country_institution_<i>_*)
# to be explicitly registered before it works — undeclared names silently
# no-op. A handful of entries here are vanilla-ship gap-fills (axes vanilla
# itself didn't auto-register).
#
# For mod-invented mechanic modifiers, see the sibling files:
#   subjugation_modifier_types.txt
#   diplomatic_play_escalation_modifier_types.txt
#   power_bloc_extra_modifier_types.txt
#   space_race_modifier_types.txt
#   nuclear_program_modifier_types.txt
#   homelands_modifier_types.txt
#   un_membership_modifier_types.txt
#   global_warming_modifier_types.txt
#   megastructure_progress_modifier_types.txt
#   banking_cycle_modifier_types.txt
#   sol_expectations_modifier_types.txt
#   (plus existing covert_warfare_, cultural_hegemony_, movement_, st_res_,
#    tech_gate_modifier_types.txt)
#
# See CLAUDE.md "Validation rules that bite if ignored" for why per-entity
# registration is mandatory.
