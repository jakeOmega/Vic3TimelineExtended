# ======================================================================
#  ENACTMENT MODIFIERS  –  used by events/extra_law_events.txt
#
#  Design principle:  enactment events come in three flavors, matching
#  vanilla — advance, cautious, stall.
#    * Advance modifiers give a strong push (+0.10/+0.15 success_add).
#    * Cautious modifiers give a small nudge (+0.02/+0.05 success_add)
#      in exchange for some other benefit (loyalists, IG approval).
#    * Stall modifiers give a sustained drag (-0.10/-0.15 success_add).
#      Paired with — or chosen against — `add_enactment_setback = 1`
#      in the corresponding stall event, so the player picks WHICH way
#      the law slows down, not whether.
#
#  When applied to an IG (not the country), the modifier represents
#  that IG's reaction: advance = pleased, cautious = disgruntled,
#  stall = openly hostile.
# ======================================================================
